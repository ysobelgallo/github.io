Practice with ggplot2!!

This sheet gives example of ggplot2 diagrams to serve as example syntax and inspiration! If you’re following along, first load the libraries we’ll need for this demonstration: ggplot2 and palmerpenguins.

library(ggplot2)
library(palmerpenguins)

Scatterplot Graph

First up, we have a scatter plot graph showing the relationship between flipper length and body mass across three penguin species.

ggplot(data=penguins) + 
  geom_jitter(mapping=aes(x=flipper_length_mm,y=body_mass_g,color=species,shape=species))+
  geom_smooth(mapping=aes(x=flipper_length_mm,y=body_mass_g),
              color="black")+
  annotate("text", x=213, y=3500, label="The Gentoos are the largest",
           color="black",fontface="bold",size=4.5,angle=25)+
  labs(title="Flipper Length and Body Mass in Palmer Penguins",
       x="Flipper Length (mm)",
       y="Body Mass (g)")
## `geom_smooth()` using method = 'loess' and formula = 'y ~ x'

Wow. That was a lot of code. Remember you can always add an echo = FALSE parameter to the code chunk to hide the R code that generated the plot. In the future, I’ll only show the code when I’ve done something particularly grand, so you can see what it is.

Oh, speaking of grand, you may have noticed the function geom_jitter. Although most scatterplots are built with geom_point, I used geom_jitter instead to help keep the data points from overlapping, and to allow for an easier read in a large data set.

Bar Charts

ggplot2 can make bar charts too! Below are two different kinds of bar charts, displaying diamond sales data. The first one is an ordinary chart showing the default count function based on parameter (x). The second chart is called a “stacked bar chart” and it shows the different variables within a set value.

As you can see, with stacked bar charts you can provide a much more complex summary of your data. Diamonds are not very interesting though, so let’s go back to penguins.

We’re going to use the same concepts above to clearly show population demographics on each island:

ggplot(data=penguins)+
  geom_bar(mapping=aes(x=island,fill=species))+
  theme(axis.text.x=element_text(angle=45))+
  labs(title="Penguin Species by Island",
       x="Island",
       y="Population")

Facet Graphs

Sometimes you may have a complicated graphic that takes too long for your audience to digest. For example, remember our first graph? The one that showed the relationship between flipper length and body mass? Just in case I’m printing it again here:

## `geom_smooth()` using method = 'loess' and formula = 'y ~ x'

In this graph, there is a clear correlation between flipper length and body mass. The longer the flippers, the larger the penguin is- and this ratio holds true across all the penguin species. There is, however, a large disparity between the actual sizes of all three penguins, and it’s possible audience members might get distracted by this. So to make the graph easier to understand, we can use the facet_wrap function:

ggplot(data=penguins) + 
  geom_jitter(mapping=aes(x=flipper_length_mm,y=body_mass_g,color=species,shape=species))+
  geom_smooth(mapping=aes(x=flipper_length_mm,y=body_mass_g),
              color="black")+
  facet_wrap(~species,scales="free")+
  labs(title="Flipper Length and Body Mass in Palmer Penguins",
       x="Flipper Length (mm)",
       y="Body Mass (g)")
## `geom_smooth()` using method = 'loess' and formula = 'y ~ x'

There. Now all the penguins have been divided out by species. I’ve left the code in so you can see how it’s done. If you’re analyzing something really complicated you can even use the facet_grid function to divide your graph into even more pieces! We’ll look at our penguin populations again, this time taking sex into account as well:

penguins_2 <- penguins %>% 
  na.omit(penguins$sex)

ggplot(data=penguins_2)+
  geom_bar(mapping=aes(x=sex,fill=sex))+
  facet_grid(species~island)

Hmm, looks like we have a fairly good sample here, with roughly equal numbers of males and females of each species represented. Notice in the code the first argument splits the chart horizontally, and the second splits the chart vertically. Notice also, I had to remove several na values from the sex column before I could graph it, using a tidyverse pipe.

ggplot2 And dplyr Combined

I’m glad we had to use the tidyverse pipe above, because it brings up a good point: you can combine packages with ggplot2!! This is very exciting. It means we can first get a subset of our data with dplyr and then visualize just that subset with ggplot2. For example, it just occurred to me, I don’t think I know whether Gentoo males or females are bigger. I’ve watched all of Blue Planet probably 10 times and somehow this fact has escaped me. I’ve picked Gentoo here, but you could really filter for anything.

library(dplyr)
penguins_Gentoo <- penguins_2 %>% 
  filter(species=="Gentoo")

ggplot(data=penguins_Gentoo)+
  geom_point(mapping=aes(x=sex,y=body_mass_g))+
  labs(title="Which Is Bigger?",
       subtitle= "A Curiosity Question About Gentoos",
       x="Sex",
       y="Body Mass (g)")

Ah ha! The males are bigger! Good to know. Notice I used the dataframe with the dropped na values from last time, to simplify the chart. I decided this was okay because there are only 4 na values here, which is not a significant amount. Also, remember to load dplyr before attempting a filter-graph sequence!! Otherwise it won’t work. You cannot take functions out of a package you haven’t opened yet. I cannot stress this enough. Asking a computer to do something like that will make it extremely, almost violently confused.
